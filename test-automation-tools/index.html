04 — Setup

セットアップの容易さ

最初の1テストを走らせるまでの手数。Playwright / Cypress はほぼ一発、Selenium はドライバ管理が加わります（Selenium Manager で近年は改善）。

Playwrightかんたん
npm init playwright@latest

ブラウザ3種・設定・サンプル・CI雛形まで一括生成。すぐ動く。

Cypressかんたん
npm i -D cypress
npx cypress open

GUIが起動し、対話的にテストを作成。開発者にやさしい導線。

Seleniumやや手数
pip install selenium
# + ドライバ管理
# (Selenium Manager が自動化)

言語別ライブラリ導入。テストランナーは別途選定が必要。

08 — FAQ

よくある質問

結局どれが一番おすすめ？＋

「唯一の正解」はありません。新規・JS/TS中心・多ブラウザなら Playwright が有力候補、フロント開発者のDX重視なら Cypress、多言語や既存資産・幅広い環境対応が要件なら Selenium が堅実です。要件から逆算して選ぶのが本ページの目的です。

Selenium はもう古い？＋

いいえ。Selenium は W3C WebDriver 標準の基盤であり、多言語対応・幅広いブラウザ/実機・膨大な資産という強みは今も健在です。Selenium Manager でドライバ管理も改善されました。モダンなDXでは新興ツールが有利な場面が多い、というのが実情です。

Cypress は Playwright に置き換わる？＋

用途次第です。Playwright はマルチブラウザ・並列・言語の幅で優位ですが、Cypress のブラウザ内実行によるタイムトラベル・デバッグ体験やコンポーネントテストを好むチームも多く、両者は共存しています。

移行や併用はできる？＋

可能です。段階的にスモークテストから新ツールへ移す、レイヤーで使い分ける（結合はSelenium、E2EはPlaywright等）といった併用も現実的です。まずは小さく試すことを推奨します。
